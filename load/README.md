# Нагрузочное тестирование (Locust)

Нагрузочные тесты для API бэкенда (Express + YDB). Имитируются типичные сценарии: просмотр мероприятий, категорий, заявок, авторизация и регистрация.

## Требования

- Python 3.8+
- Бэкенд доступен по URL (локально или на сервере)

## Установка

На macOS часто доступен только `pip3`. Используйте один из вариантов:

```bash
cd load
pip3 install -r requirements.txt
```

Если `pip3` не найден — через Python:

```bash
python3 -m pip install -r requirements.txt
```

## Запуск

**Если бэкенд уже на сервере** и его URL прописан в корневом `.env` (`VITE_API_BASE_URL` или `API_BASE_URL`), достаточно запустить Locust из папки `load` — хост подставится из .env автоматически:

```bash
cd load
python3 -m locust -f locustfile.py
```

(Команда `locust` может быть недоступна в PATH — тогда всегда используйте `python3 -m locust`.)

Откройте в браузере http://localhost:8089, задайте число пользователей и RPS, затем запустите тест.

**Явно указать хост** (переопределяет .env):

```bash
python3 -m locust -f locustfile.py --host=http://your-server:3000
```

**Без веб-интерфейса (headless):**

```bash
# 10 виртуальных пользователей, 2 в секунду, длительность 30 секунд
python3 -m locust -f locustfile.py --headless -u 10 -r 2 -t 30s
```

## Параметры

| Параметр | Описание |
|----------|----------|
| `-u, --users` | Число виртуальных пользователей |
| `-r, --spawn-rate` | Скорость появления пользователей (в секунду) |
| `-t, --run-time` | Время работы теста (например `30s`, `5m`, `1h`) |
| `--host` | Базовый URL API (обязательно) |

## Сценарии в locustfile

- **GET /events** — список мероприятий (наибольший вес)
- **GET /events** с фильтрами
- **GET /categories**
- **GET /users**, **GET /applications**, **GET /eventRequests**, **GET /notifications**
- **POST /auth** — попытки входа
- **POST /auth/register** — регистрация новых пользователей

Веса задач заданы так, чтобы чаще нагружать чтение каталога мероприятий и категорий, реже — запись (логин/регистрация).
